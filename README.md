# school_love_pair_system

## 项目简介
本项目基于**Diffie-Hellman密钥交换**、**ElGamal乘法同态加密**、**国密SM3/SM4**算法，实现隐私保护型网恋匹配系统。
平台无法获知用户选择与匹配结果，仅匹配双方知晓最终结果；拒绝方无法得知对方选择，严格保护用户隐私。

## 环境依赖
```bash
pip install gmssl
pip install pycryptodome
```

## 运行步骤
1. 运行服务器端：`Server.py`
```bash
python Server.py
```
2. 运行客户端：`Client.py`（可多开）
```bash
python Client.py
```
4. 客户端操作：注册 → 登录 → 完善信息 → 点击匹配

## 系统架构
- 服务器端：连接管理、用户认证、匹配调度、密文同态计算
- 客户端：GUI交互、密钥生成、加密/解密、结果显示
- 密码模块：ElGamal、SM3、SM4
- 数据库：SQLite存储用户信息、匹配条件筛选

## 文件说明
- `server.py`：服务器主程序
- `Client.py`：客户端GUI程序
- `ElGamal.py`：ElGamal加密、解密、同态乘法
- `sm3.py`：国密SM3哈希、密钥派生
- `data.py`：数据库初始化、用户查询、匹配逻辑
- `client_information.db`：用户信息数据库


## 核心安全特性
1. **Diffie-Hellman 密钥协商安全**
   - 基于离散对数难题，双方在不安全信道上生成共享密钥
   - 服务器仅中转公钥，无法获取共享密钥
   - 私钥本地保存，永不上传，杜绝密钥泄露

2. **ElGamal 同态加密隐私计算**
   - 采用乘法同态特性，服务器仅对密文做运算，不解密、不获知原始选择
   - 接受加密为 1，拒绝加密为随机数，密文不可区分，保护单端选择隐私
   - 仅当双方都接受，同态乘积才可解密为有效结果

3. **密文再随机化防推导**
   - 服务器对同态乘积结果进行随机化处理
   - 防止用户通过结果反推对方选择，实现“拒绝方不知对方态度”

4. **国密算法全流程加固**
   - SM3 用于密码哈希、密钥派生，抗碰撞性强
   - SM4 对称加密电话号码等敏感信息
   - 全程使用国产密码算法，安全合规、自主可控

5. **单向选择不可见性**
   - 若一方拒绝，无法得知对方是否接受
   - 避免单向喜欢带来的尴尬与隐私泄露
   - 匹配结果仅双方可见，服务器完全盲计算

6. **敏感数据加密存储与传输**
   - 联系方式（电话）本地 SM4 加密后上传
   - 仅匹配成功方可解密获取联系方式
   - 数据库仅存储密文，防止数据拖库泄露

7. **双向匹配与防重放安全**
   - 严格双向推送，确保匹配公平性
   - 本地记录已推送列表，避免重复推送与攻击
   - 离线用户上线后可自动获取历史匹配结果

8. **服务器最小权限设计**
   - 服务器不存储任何私钥、共享密钥
   - 不参与解密，仅做消息中转与密文计算
   - 即使服务器被攻破，也无法还原用户选择

9. **身份认证与防篡改**
   - 密码经 SM3 哈希存储，防明文泄露
   - 公钥交换前完成身份认证
   - 消息带固定前缀，防伪造、防重放、防注入

10. **鲁棒安全与异常防护**
    - 私钥文件损坏自动重建，不影响安全
    - 输入合法性校验，防止恶意数据攻击
    - 多线程隔离，客户端独立安全上下文
