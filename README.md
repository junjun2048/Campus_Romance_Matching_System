# 校园恋爱匹配系统

## 项目简介
本项目基于Diffie‑Hellman 密钥交换、ElGamal 乘法同态加密以及国密 SM3/SM4 密码算法，构建了一套面向校内场景、强隐私保护的网恋匹配系统。系统以 “平台不可信、数据不透明、选择不泄露” 为安全目标，在服务器仅承担转发、存储与密文计算的半可信模型下，实现全程隐私保护。用户在注册、资料填写、双向匹配、接受 / 拒绝等全流程中，所有敏感选择与匹配结果均以密文形式提交与处理，服务器无法解密、无法窥探、无法推断任何用户的真实偏好与最终匹配状态。只有匹配双方能够通过本地共享密钥解密得知结果，并且能够得到对方的联系方式；若任意一方拒绝，则双方均无法获知对方选择，从密码学层面杜绝单向暴露、尴尬泄露与隐私偷窥，真正实现 “平台看不见、对方猜不到、结果只有自己知道” 的安全匹配机制，为公钥密码在隐私计算、社交安全、匿名匹配等场景提供可落地的实践方案。

## 运行步骤
1. 运行服务器端：`Server.py`
```bash
python Server.py
```
2. 运行客户端：`Client.py`，可运行多个
```bash
python Client.py
```

## 系统架构
### 1. 服务器端（server.py）
- 负责客户端连接管理、用户认证、用户信息存储
- 实现双向匹配推送、密文同态计算、历史匹配结果缓存
- 提供管理员控制台，支持在线用户查看、消息发送、服务器关闭

### 2. 客户端（Client.py）
- 基于 Tkinter 实现图形化交互界面
- 完成密钥生成、Diffie-Hellman 共享密钥计算
- 实现选择加密、结果解密、联系方式解密展示

### 3. 密码算法模块
- ElGamal：加密、解密、乘法同态计算
- SM3：密码哈希、密钥派生
- SM4：用户联系方式对称加密

### 4. 数据库模块（data.py）
- 基于 SQLite 轻量级数据库
- 存储用户信息、公钥、匹配条件筛选

## 核心功能
1. 用户注册、登录、密码 SM3 哈希安全存储
2. 个人信息完善与验证（年龄、电话格式校验）
3. 异性智能匹配（年龄差 ±5 岁）
4. 双向推送机制，确保推送公平对等
5. 基于 DH 密钥交换的安全信道建立
6. ElGamal 同态加密实现隐私匹配
7. 拒绝方无法获知对方选择，避免尴尬
8. 联系方式 SM4 加密，仅匹配成功可解密
9. 离线用户上线自动获取历史匹配结果
10. 服务器全程盲计算，无法窥探任何隐私

## 核心安全特性
1. **Diffie-Hellman 密钥协商安全**
   - 基于离散对数难题，双方在不安全信道生成共享密钥
   - 服务器仅中转公钥，无法获取共享密钥
   - 私钥本地保存，永不上传，杜绝密钥泄露

2. **ElGamal 同态加密隐私计算**
   - 采用乘法同态特性，服务器仅对密文运算，不解密、不获知原始选择
   - 接受加密为 1，拒绝加密为随机数，密文不可区分
   - 仅当双方都接受，同态乘积才可解密为有效结果

3. **密文再随机化防推导**
   - 服务器对同态乘积结果进行随机化处理
   - 防止用户通过结果反推对方选择，实现单向选择不可见

4. **国密算法全流程加固**
   - SM3 用于密码哈希、密钥派生，抗碰撞性强
   - SM4 对称加密电话号码等敏感信息
   - 全程使用国产密码算法，安全合规、自主可控

5. **单向选择不可见性**
   - 若一方拒绝，无法得知对方是否接受
   - 避免单向喜欢带来的尴尬与隐私泄露
   - 匹配结果仅双方可见，服务器完全盲计算

6. **敏感数据加密存储与传输**
   - 联系方式（电话）本地 SM4 加密后上传
   - 仅匹配成功方可解密获取联系方式
   - 数据库仅存储密文，防止数据拖库泄露

7. **双向匹配与防重放安全**
   - 严格双向推送，确保匹配公平性
   - 本地记录已推送列表，避免重复推送与攻击
   - 离线用户上线后自动获取历史匹配结果

8. **服务器最小权限设计**
   - 服务器不存储任何私钥、共享密钥
   - 不参与解密，仅做消息中转与密文计算
   - 即使服务器被攻破，也无法还原用户选择

9. **身份认证与防篡改**
   - 密码经 SM3 哈希存储，防明文泄露
   - 公钥交换前完成身份认证
   - 消息带固定前缀，防伪造、防重放、防注入

10. **鲁棒安全与异常防护**
    - 私钥文件损坏自动重建，不影响安全
    - 输入合法性校验，防止恶意数据攻击
    - 多线程隔离，客户端独立安全上下文

## 联系方式获取流程（核心）
1. 用户提交信息时，客户端生成 SM4 密钥加密电话号码，密钥本地保存
2. 匹配时双方通过 DH 协商共享密钥，用于加密选择与联系方式密钥
3. 用户选择以 ElGamal 密文提交，服务器进行同态乘法计算
4. 仅当双方均接受，解密结果为对方 SM4 密钥
5. 用 SM4 密钥解密得到对方真实电话号码
6. 任意一方拒绝，均无法得到任何有效信息
